# Contributing

Project Master values correctness over activity volume.

Before submitting a change:

1. State the problem precisely.
2. Distinguish measured behavior from assumptions.
3. Add or update tests.
4. Explain how the change affects epistemic reliability.
5. Document new permissions or security implications.
6. Avoid adding a capability without a verification path.

Run:

```powershell
ruff check .
pytest
```
